> 由于微软的历史原因，power shell 7以上的版本还是叫做powershell，但是实际可执行文件名字叫做：pwsh.exe，因此以后我们用到power shell7都成为pwsh



## 为什么要用pwsh

1. **历史记录**
2. **智能补全（与Unix终端类似的体验）**
3. 跨平台（我们公司暂时用不到）

想要达到以上效果，可以这么配置

```bash
# 1. 确保配置文件路径存在
New-Item -Path (Split-Path $profile) -ItemType Directory -Force

# 2. 创建文件
New-Item -Path $profile -ItemType File -Force

# 3. 打开文件
notepad $profile

```

然后输入以下内容

```bash
# ==================== PSReadLine 完整配置 ====================

# ---------- 基础选项 ----------
Set-PSReadLineOption `
    -PredictionSource History `
    -PredictionViewStyle ListView `
    -BellStyle None `
    -CompletionQueryItems 100 `
    -ShowToolTips:$false `
    -HistorySaveStyle SaveIncrementally `
    -HistoryNoDuplicates:$true

# ---------- 快捷键绑定 ----------

# Tab 键：菜单补全
Set-PSReadLineKeyHandler -Chord Tab -Function MenuComplete

# 上下箭头：智能历史搜索 + 自动移动到行尾
Set-PsReadLineKeyHandler -Key UpArrow -ScriptBlock {
    [Microsoft.PowerShell.PSConsoleReadLine]::HistorySearchBackward()
    [Microsoft.PowerShell.PSConsoleReadLine]::EndOfLine()
}

Set-PSReadLineKeyHandler -Key DownArrow -ScriptBlock {
    [Microsoft.PowerShell.PSConsoleReadLine]::HistorySearchForward()
    [Microsoft.PowerShell.PSConsoleReadLine]::EndOfLine()
}

# Ctrl+Z：撤销操作
Set-PSReadLineKeyHandler -Chord Ctrl+z -Function Undo

# Ctrl+D：删除字符或退出会话（类 Unix 行为）
Set-PSReadlineKeyHandler -Chord Ctrl+d -Function DeleteCharOrExit
```


## 原理

以上都是PSReadLine的能力，至于其说明书，可以参考：[使用 PSReadLine 键处理程序 - PowerShell | Microsoft Learn](https://learn.microsoft.com/zh-cn/powershell/scripting/learn/shell/using-keyhandlers?view=powershell-7.6)

里面提到了Get-PSReadLineKeyHandler，在pwsh中运行这个命令，即可看到所有帮助和选项

![alt text](/assets/img/post/2026-03-37-pwsh-01/img-6.png)

接下来，我会详细解释每一行命令的内容，没兴趣的可以不看

---

这些是 PowerShell 7 中用于配置 PSReadLine 模块的命令，PSReadLine 负责控制命令行编辑、历史记录和补全体验。逐条解释如下：

### 1. Set-PSReadLineOption -PredictionSource HistoryAndPlugin

作用：设置预测建议的来源

HistoryAndPlugin：同时从以下两个来源获取预测：

History：基于你之前输入过的命令历史

Plugin：允许外部插件提供预测（如 AI 预测、自定义预测器等）

效果：当你输入命令时，灰色文本会显示预测的命令，按 → 键即可接受建议

### 2. Set-PSReadLineKeyHandler -Chord Tab -Function MenuComplete

作用：将 Tab 键 的行为从默认的“标准补全”改为“菜单补全”

MenuComplete：按 Tab 会弹出一个可导航的补全菜单，用方向键选择，而不是直接循环补全

效果：更适合快速浏览和选择命令、路径、参数等补全项

### 3. Set-PsReadLineKeyHandler -Key UpArrow -ScriptBlock { ... }

作用：自定义 上箭头键 的行为

脚本块：

HistorySearchBackward()：在历史记录中向后搜索（与当前输入内容匹配的历史命令）

EndOfLine()：将光标移动到行尾

效果：按上箭头时，会基于当前已输入的内容搜索匹配的历史命令，并自动将光标移到行尾，方便直接修改或执行

### 4. Set-PSReadLineKeyHandler -Key DownArrow -ScriptBlock { ... }

作用：自定义 下箭头键 的行为

脚本块：

HistorySearchForward()：在历史记录中向前搜索

EndOfLine()：将光标移动到行尾

效果：按上箭头搜索历史后，可以用下箭头返回更近的命令，光标始终在行尾

### 5. Set-PSReadLineOption -PredictionViewStyle ListView

作用：设置预测建议的显示样式

ListView：预测建议以列表视图形式显示在光标下方（垂直列表），而不是默认的内联灰色文本

效果：更直观，可以用上下箭头在多个预测建议之间选择

### 整体体验总结

这套配置让你的 PowerShell 7 更接近 现代 shell 体验（类似 zsh 或 fish）：

基于历史+插件的智能预测

Tab 键弹出可选菜单

上下箭头智能搜索历史

预测建议以列表形式清晰呈现

如果希望立即生效，将这些命令添加到 PowerShell 配置文件（如 $PROFILE）中即可。
