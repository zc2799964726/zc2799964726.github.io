## 一、使用vcpkg进行全局安装（不推荐）

询问Gemini,给我提供了使用vcpkg的方法安装，相对比较省心，因此我就直接开始行动,中间也发生了一些小问题，因此现在记录一下正确步骤

首先在c:/dev下克隆vcpkg仓库

```bash
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
# 运行引导脚本 (Bootstrap)
./bootstrap-vcpkg.bat
```

然后vcpkg全局安装Ceres库，编译完成后，所有生成的文件统一放在 vcpkg 安装根目录下的 installed 文件夹中

```bash
./vcpkg install ceres:x64-windows
```

## 二、随项目走而非全局配置

我的需求是 "所有3方仓库放在项目根目录下的 3rdParty 下"，并让 find_package 找到它们

vcpkg 默认是全局安装，有两种方法可以实现我的需求，详细介绍方案一清单模式

### 方案一：vcpkg "清单模式" (Manifest Mode) + CMake 集成

这是最现代、最推荐的方法。它结合了 vcpkg 的易用性 和 我 "依赖随项目走" 的需求

1. 在我的项目根目录 (与 3rdParty 同级) 创建一个 vcpkg.json 文件（后续可以删掉）

```json
{
  "$schema": "https://raw.githubusercontent.com/microsoft/vcpkg-tool/main/docs/vcpkg.schema.json",
  "name": "scan3d",
  "version-string": "1.0.0",
  "dependencies": [
    {
      "name": "ceres",
      "features": ["suitesparse"]
    }
  ]
}
```

2. 使用清单模式命令安装 Ceres：

```bash
# 注意：使用 vcpkg.exe 而不是 .\vcpkg
# <my-project-path-root>需要替换成与上述 vcpkg.json 相同的路径
C:\dev\vcpkg\vcpkg.exe install --triplet x64-windows --host-triplet x64-windows --x-manifest-root=<my-project-path-root>
```

运行情况如下截图（网络要好哦，下载了18个依赖的包），只截图最后部分

![安装ceres](/assets/img/post/2026-05-02-BA-03/image3.png)

项目中会出现 vcpkg_installed这个目录

![安装ceres](/assets/img/post/2026-05-02-BA-03/image4.png)


3. 配置我的主 CMakeLists.txt 以使用 vcpkg 工具链：

- vcpkg 安装在 C:/dev/vcpkg（可以安装到别的位置）
- 第一次配置 CMake 时（例如在命令行或 CLion/VS Code 的设置中），需要指定 CMAKE_TOOLCHAIN_FILE

```bash
# 假设你在一个 'build' 文件夹中
# 记得用一个空的 build 目录
cmake .. -DCMAKE_TOOLCHAIN_FILE=C:/dev/vcpkg/scripts/buildsystems/vcpkg.cmake
```

![安装ceres](/assets/img/post/2026-05-02-BA-03/image5.png)

重要提示： 如果你之前在 build 目录中运行过不带 CMAKE_TOOLCHAIN_FILE 的配置，请删除 D:\git\scan3d\build 目录中的所有内容（包括 CMakeCache.txt），然后重新运行上述完整的 cmake .. -DCMAKE_TOOLCHAIN_FILE=... 命令

否则一定会出现各种失败的问题。

4. vcpkg 的工作方式：

- 当 CMake 运行时，vcpkg 工具链会读取你的 vcpkg.json 文件。

- 它会自动下载并编译 ceres 到你项目的构建目录下的一个 vcpkg_installed 文件夹中。

- 它会自动配置 find_package 所需的搜索路径。


5. CMakeLists.txt的改动

```cmake
cmake_minimum_required(VERSION 3.15)
project(MyCeresProject)

# 你不需要 set(CMAKE_PREFIX_PATH ...)
# vcpkg 工具链已经帮你完成了！
find_package(Ceres REQUIRED)

add_executable(my_app main.cpp)

# Ceres::ceres 目标已经可用
target_link_libraries(my_app PRIVATE Ceres::ceres)
```
