## 1. 认识BA

Bundle Adjustment, 也叫光束调整，在测绘领域有另外一个名称：光束法平差

直接说可能比较抽象，我们先从最简单的例子来说吧：直线拟合

---

### 一、从最简单的问题开始：直线拟合

假设我们想找到一条直线：

```
y = ax + b
```

其中：

```
a = 斜率
b = 截距
```

我们要做的事情是：

**根据一组测量数据，估计参数**

```
[a, b]^T
```

---

### 二、如果测量没有误差

假设真实直线是：

```
y = x + 2
```

即

```
a = 1
b = 2
```

如果测量完全准确：

```
(x1,y1) = (1,3)
(x2,y2) = (2,4)
```

只需要 **两组点** 就可以解出：

```
a = 1
b = 2
```

因为两个未知数，两条方程即可。

---

### 三、现实世界的问题：测量误差

现实中数据通常会有误差，例如：

```
(x1,y1) = (1,3.1)
(x2,y2) = (2,4.1)
(x3,y3) = (3,4.9)
```

如果仍然用

```
y = ax + b
```

去解释这些数据，会发现：

**没有一条直线能够完全通过所有点。**

如下图的直觉：

```
真实直线
  |
  |    *
  |  *
  |      *
```

点会 **围绕直线分布**。

---

### 四、模型预测 vs 实际观测

对于每一个点：

真实观测：

```
yi
```

直线模型预测：

```
ŷi = axi + b
```

两者的差值称为：

```
残差 residual
```

定义：

```
ri = yi − (axi + b)
```

直观理解：

```
残差 = 实际点 到 直线 的垂直偏差（近似）
```

例如你给的例子：

真实参数：

```
[a,b] = [1,2]
```

数据：

```
(1,3.1)
(2,4.1)
(3,4.9)
```

预测：

```
ŷ1 = 1*1+2 = 3
ŷ2 = 4
ŷ3 = 5
```

残差：

```
r1 = 3.1 − 3 = 0.1
r2 = 4.1 − 4 = 0.1
r3 = 4.9 − 5 = -0.1
```

---

### 五、目标：找到“最合理”的直线

由于测量存在误差，我们不能要求：

```
ri = 0
```

但我们希望：

```
所有残差尽可能小
```

因此定义一个目标函数：

```
残差平方和
```

即：

```
F(a,b) = r1² + r2² + ... + rn²
```

展开：

```
F(a,b) = Σ (yi − (axi + b))²
```

我们的目标就是：

```
min F(a,b)
```

也就是：

```
min Σ (yi − (axi + b))²
```

---

### 六、为什么要平方？

平方有三个重要原因：

#### 1 避免正负抵消

如果直接求和：

```
0.1 + (-0.1) = 0
```

误差会互相抵消。

平方之后：

```
0.01 + 0.01
```

不会抵消。

---

#### 2 大误差惩罚更大

平方会让大的误差权重更高：

```
0.1² = 0.01
0.5² = 0.25
```

---

#### 3 数学上容易求解

平方后函数是：

```
光滑可导
```

可以用：

```
梯度下降
高斯牛顿
LM
```

等方法求解。

---

### 七、最终问题形式

直线拟合最终变成一个 **最小二乘问题**：

```
min Σ (yi − (axi + b))²
```

其中：

```
参数：a,b
数据：xi,yi
```

通过调整参数

```
[a,b]
```

让模型

```
y = ax + b
```

**尽可能接近所有观测点。**

---

### 八、和 Bundle Adjustment 的关系

直线拟合其实是一个 **最简单的非线性最小二乘问题**。

而
**Bundle Adjustment**

本质也是一样的：

只不过：

| 问题     | 参数              |
| -------- | ----------------- |
| 直线拟合 | a,b               |
| BA       | 相机位姿 + 三维点 |

残差也类似：

直线拟合：

```
ri = yi − (axi + b)
```

BA：

```
ri = observed_pixel − projected_pixel
```

目标函数完全一样：

```
min Σ ||ri||²
```

唯一变化：

```
参数规模变大
模型更复杂
```

---

### 九、一个更完整的逻辑总结

从简单到复杂可以这样理解：

```
直线拟合
   ↓
最小二乘
   ↓
非线性最小二乘
   ↓
Bundle Adjustment
```

Bundle Adjustment 其实就是：

```
超大规模的最小二乘问题
```

只不过参数是：

```
camera pose
3D points
```

而残差是：

```
reprojection error
```


## 2. Ceres库简单使用

对于上述直线拟合，我们就可以使用Ceres库进行计算

```c++
#include <ceres/ceres.h>
#include <iostream>

struct LineCost
{
    LineCost(double x, double y) : x_(x), y_(y) {}

    template<typename T>
    bool operator()(const T* const a,
                    const T* const b,
                    T* residual) const
    {
        residual[0] = T(y_) - (a[0]*T(x_) + b[0]);
        return true;
    }

    double x_, y_;
};

int main()
{
    double a = 0.0;
    double b = 0.0;

    ceres::Problem problem;

    double data[5][2] = {
        {1,2},{2,4},{3,6},{4,8},{5,10}
    };

    for(auto& d : data)
    {
        ceres::CostFunction* cost =
            new ceres::AutoDiffCostFunction<LineCost,1,1,1>(
                new LineCost(d[0],d[1]));

        problem.AddResidualBlock(cost,nullptr,&a,&b);
    }

    ceres::Solver::Options options;
    options.minimizer_progress_to_stdout=true;

    ceres::Solver::Summary summary;

    ceres::Solve(options,&problem,&summary);

    std::cout<<"a="<<a<<" b="<<b<<std::endl;
}
```

其核心结构就是

```
CostFunction
Problem
Solve
```
